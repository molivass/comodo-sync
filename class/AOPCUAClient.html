AOPCUAClient

Hereda: AActor

Clase principal del plugin.

Descripción

Contiene todas las funciones de conexión a servidores OPC UA, lectura y escritura de nodos, y gestión de suscripciones.

Actúa a modo de UBlueprintFunctionLibrary, pero se define como AActor para poder pasar el objeto como contexto en las funciones, ya que la librería open62541 requiere de un contexto para monitorizar. Por defecto Unreal pasa self como contexto en blueprints, por lo que no afecta a la forma de uso.

Delegates

Dynamic FOPCUAOnValueChanged

Delegate dinámico simple.

Ejecuta el evento vinculado cada vez que recibe un cambio de valor de un nodo monitorizado. Ver DataChangeNotificationCallback() para más detalles.

Descripciones de Propiedades

bool bIsConnected = false

Devuelve si el cliente está conectado a un servidor o no. Client debe estar inicializado para poder conectarse.

UA_Client * Client = nullptr

Puntero al cliente de open62541.

Se inicializa al conectar a un servidor con ConnectAnon() o ConnectUser(), y se destruye al desconectarse con Disconnect.

TMap<FString, FMonitoredItem> MonitoredItems = {}

Mapa que almacena los nodos que se están monitorizando. Un nodo monitorizado envía notificaciones al cambiar de valor, lo que permite reaccionar a esos cambios en tiempo real.

La clave es es un FString con el formato NodeIdString:Namespace

El valor es una estructura FMonitoredItem que contiene los datos referentes al nodo monitorizado. Ver la definicón de la estructura para más detalles.

Descripciones de Métodos

void AOPCUAClient()

Constructor de la clase.

Establece bIsConnected a false y Client a nullptr.

void BeginPlay() virtual override

Función llamada al iniciar el juego o al spawnear el actor.

Simplemente llama al Super::BeginPlay().

void EndPlay(const EEndPlayReason::Type EndPlayReason) virtual override

Función llamada al finalizar el juego o al destruir el actor.

Llama a Disconnect() para asegurarse de que el cliente se desconecta del servidor, y se eliminan las suscripciones. Luego llama al Super::EndPlay(EndPlayReason).

void Tick(float DeltaTime) virtual override

Función llamada cada frame.

Llama al Super::Tick(DeltaTime).

Si IsConnected() devuelve true, es decir, si el cliente está conectado a un servidor, entonces ejecuta ProcessClientEvents.

bool ConnectAnon(const FString URL)

Conecta a un servidor OPC UA de forma anónima, sin credenciales de usuario.

Advertencia

El servidor al que se conecta debe permitir conexiones anónimas, de lo contrario la conexión fallará.

  • URL: Es la dirección del servidor, en el formato opc.tcp://IP:PUERTO. Por ejemplo, opc.tcp://192.168.1.100:4840.

Establece Client con un nuevo cliente de open62541, e intenta conectar al servidor de forma anónima. Si se conecta correctamente, establece bIsConnected a true. Si ocurre un error, destruye el cliente, estableciendo Client a nullptr. Devuelve true si la conexión se establece correctamente, o false si ocurre un error.

bool ConnectUser(const FString URL, const FString Username, const FString Password)

Conecta a un servidor OPC UA con usuario y contraseña.

Error

Actualmente no funciona y siempre retornará false

  • URL: Es la dirección del servidor, en el formato opc.tcp://IP:PUERTO. Por ejemplo, opc.tcp://192.168.1.100:4840.

  • Username: El nombre de usuario definido en el servidor

  • Password: Contraseña asociada al usuario.

Establece Client con un nuevo cliente de open62541, e intenta conectar al servidor con las credenciales proporcionadas. Si se conecta correctamente, establece bIsConnected a true. Si ocurre un error, destruye el cliente, estableciendo Client a nullptr.

Devuelve true si la conexión se establece correctamente, o false si ocurre un error.

void Disconnect()

Desconecta el cliente del servidor.

Llama a StopAllMonitoring() para eliminar las posibles suscripciones, y se desconecta del servidor en caso de Client estuviera definido.

Establece Client a null y bIsConnected a false.

bool IsConnected()

Devuelve el estado de conexión al servidor.

Devuelve true si Client está definido, y bIsConnected es true

Definición para blueprints de ReadNodeValueInternal().

Importante

No se debe llamar a esta función directamente. En Blueprints, se ejecuta execReadNodeValue(), que cambia el tipo de OutValue y pasa los valores a ReadNodeValueInternal().

En c++ llamar directamente a ReadNodeValueInternal() con los parámetros necesarios.

execReadNodeValue()

Ejecución interna de ReadNodeValue().

Importante

La función se ejecuta directamente al llamar a ReadNodeValue() desde Blueprints. Llamar a la función desde c++ puede dar resultados inesperados. Se recomienda llamar a ReadNodeValueInternal().

Revisa el stack para obtener la dirección de memoria de OutValue, y su FProperty, y se los pasa junto al resto de parámetros a ReadNodeValueInternal().

bool ReadNodeValueInternal(const FString& NodeIdString, int32 Namespace, void* OutValue, FProperty* OutputProperty)

Lee el valor actual de un nodo definido en el servidor.

  • NodeIdString: El nombre del nodo en el servidor, del cual se quiere leer su valor

  • Namespace: El namespace en el que se encuentra el nodo en el servidor

  • OutValue: Pin de salida que devuelve el valor obtenido del servidor.

  • OutputProperty: Tipo de variable de OutValue. Usado internamente para hacer cast al pin de salida al tipo correspondiente.

La función solo se ejecuta si existe una conexión con un servidor.

Solicita el valor al servidor, hace un cast al tipo correspondiente, y lo devuelve por OutValue.

Nota

Actualmente, el tipo de OutValue y del nodo del servidor deben coincidir exactamente. Nótese que en Blueprints las variables de tipo float son de doble precisión, es decir, el nodo del servidor ha de ser de tipo double o equivalente para poder comunicarse.

Devuelve true si se ha conseguido leer el nodo y OutValue retorna un valor, o false si ocurre un error, y OutValue es null

bool WriteNodeValueInternal(const FString& NodeIdString, int32 Namespace, void* InValue, FProperty* InputProperty)

Escribe el valor que se pasa como parámetro en un nodo del servidor.

  • NodeIdString: El nombre del nodo en el servidor, el cual se quiere cambiar de valor

  • Namespace: El namespace en el que se encuentra el nodo en el servidor

  • InValue: Valor al que se quiere establecer el nodo.

  • InputProperty: Tipo de variable de InValue. Usado internamente para hacer cast al pin de entrada al tipo correspondiente.

La función solo se ejecuta si existe una conexión con un servidor.

Hace cast del valor al tipo equivalente de OPC UA, e intenta escribirlo en el nodo.

Nota

Si el tipo de InValue no coincide con el tipo del nodo del servidor, la escritura fallará. Nótese que en Blueprints las variables de tipo float son de doble precisión, es decir, el nodo del servidor ha de ser de tipo double o equivalente para poder comunicarse.

Devuelve true si se ha escrito correctamente, o false si ocurre un error.

bool StartMonitoringNodeInternal(const FString& NodeIdString, int32 Namespace, float SamplingInterval, void* InValue, FProperty* InputProperty, FOPCUAOnValueChanged Callback)

Inicia la monitorización de un nodo del servidor. Actualiza la variable que se pasa como parámetro automáticamente cada vez que el nodo cambia de valor en el servidor.

  • NodeIdString: El nombre del nodo del servidor que se quiere monitorizar

  • Namespace: El namespace en el que se encuentra el nodo en el servidor

  • SamplingInterval: Intervalo en milisegundos entre peticiones al servidor para recibir actualizaciones.

  • InValue: Variable que la función actualizará cada vez que reciba un nuevo valor del servidor.

  • InputProperty: Tipo de variable de InValue. Usado internamente para hacer cast al pin de entrada al tipo correspondiente.

  • Callback: Delegate que se ejecutará tras cambiar el valor de InValue. Conectar el pin en Blueprints es opcional.

La función solo se ejecuta si existe una conexión con un servidor.

Si ya se está monitorizando el nodo, la función devuelve false con un mensaje indicativo.

Crea una suscripción y comienza la monitorización del nodo. Se pasa la función DataChangeNotificationCallback() al crear la monitorización, que se ejecuta cada vez que se recibe un cambio del servidor. Almacena la referencia a la monitorización en MonitoredItems

La función devuelve true si se ha empezado a monitorizar el nodo correctamente.

bool StopMonitoringNode(const FString& NodeIdString, int32 Namespace)

Detiene la monitorización de un nodo.

  • NodeIdString: El nombre del nodo en el servidor, el cual se quiere dejar de monitorizar

  • Namespace: El namespace en el que se encuentra el nodo en el servidor

Si el nodo no existe en MonitoredItems, la función devuelve false. En caso contrario, intenta eliminar la monitorización del nodo. Si no hay más nodos monitorizados bajo la misma suscripción, también elimina la suscripción. Finalmente elimina la referencia a la monitorización de MonitoredItems.

Devuelve true si se ha terminado la monitorización correctamente, o false si hubo un error.

void StopAllMonitoring()

Llama a StopMonitoringNode() para cada elemento de MonitoredItems.

  • Namespace: El namespace en el que se encuentra el nodo en el servidor

void DataChangeNotificationCallback(UA_Client* client, UA_UInt32 subId, void* subContext, UA_UInt32 monId, void* monContext, UA_DataValue* value) static __cdecl

Función interna. Es llamada automáticamente al recibir un cambio de valor de un nodo monitorizado.

  • Client: El cliente conectado al servidor. Client

  • subId: Id de suscripción. FMonitoredItem.SubscriptionId

  • subContext: Contexto de la suscripción. Debe ser un AOPCUAClient this en c++, self en Blueprints llamando desde un actor AOPCUAClient.

  • monId: Id de la monitorización. FMonitoredItem.MonitoredItemId

  • monContext: Contexto de la monitorización. No se usa actualmente, puede ser NULL

  • value: El nuevo valor que retorna el nodo del servidor.

La función solo se ejecuta si se ha pasado un contexto de subscripción válido, de clase AOPCUAClient, y el puntero de value no es nulo.

Busca un elemento en MonitoredItems cuyas id de subscripción y monitorización coincidan con subId y monId respectivamente. Si existe, intenta hacer cast a la variable FMonitoredItem.VarRef al mismo tipo que el nodo del servidor,

y establece su valor al nuevo recibido en value.

Nota

Si el tipo de la variable no coincide con el tipo del nodo del servidor, la escritura fallará. Nótese que en Blueprints las variables de tipo float son de doble precisión, es decir, el nodo del servidor ha de ser de tipo double o equivalente para poder comunicarse.

Tras actualizar la variable, lanza el delegate de FMonitoredItem.Callback.

void ProcessClientEvents()

La función solo se ejecuta si Client está definido.

Llama a UA_client_run_iterate para asegurar que los eventos asíncronos y otras tareas de la librería se ejecutan. Mantiene la conexión con el servidor estable.
